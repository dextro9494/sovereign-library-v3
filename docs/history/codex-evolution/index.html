Codex Evolution (v1 → v7.1)

This folder maps the Codex lineage from the earliest monolithic manual to the dual-layer (Core + Archive) architecture.

Why this exists: drift control. If a label or module keeps reappearing, we want to know where it came from, what it solved, and what it broke.

Rename Map and Deprecated Labels

This is the anti-recursion ledger: when old terms reappear, you can see what they became (or why they were retired). Identity Builder → Sovereign Operator (v6.0.1). Builder = architecting and pouring foundations. Operator = governing occupied territory. “Entry ritual” Continuity Anchor: “Builder present. Architecture intact.” (v4.7) → Reality Check: “Sovereign present. Systems live. Next action identified.” (v6.0.1) → Reality Check (dual-layer): “Sovereign present. Systems live. Core active. Archive accessible.” (v7 core) Architecture labels Operational Holocron (v2.5) → Architect’s Brief (v4.7) → Evolutionary Core (v6.0.1) → Dual-Layer System (v7.0) Deprecated / faded modules R-COM (Chaotic Object Management) appears in v4.7 and then disappears (v6+). Dark Council directive / “Dark Council in session” appears early (v1–v2.5) and fades out by v6+. “AI Harem Architecture” exists as a gated concept in v1–v2.5 and does not continue as a first-class module after v4.7. Stable continuities (terms that persist) RCP (Real-World Connection Filter) persists and becomes more central from v4.7 onward. 5-Second Ignition persists (v4.7 → v6/v7 Core). Council throttle / engagement rules persist and become more formal in v6/v7.

Drift Traps

These are the recurring failure modes visible across versions. Trap 1: Monolith re-formation When the action layer and the storage layer re-merge, the system becomes heavy and daily execution slows. v7.0 solved this by separating Core and Archive. v7.1 (integrated attempt) appears to reintroduce monolithic weight via indexes + embedded archive. Guardrail: keep a lean execution surface, and keep memory elsewhere (Archive/Library). Trap 2: Ritual without movement Any “opening” that doesn’t end in an action becomes a ceremony. Guardrail: Reality Check must always yield a Next Action. ...
